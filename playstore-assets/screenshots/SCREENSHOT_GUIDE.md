# Screenshot Guide for Google Play Store

## Requirements

### Phone Screenshots (Required)
- **Dimensions:** 1080 x 1920 pixels (portrait)
- **Minimum:** 2 screenshots
- **Maximum:** 8 screenshots
- **Format:** PNG or JPG (24-bit, no alpha)

## Screenshot Best Practices

### Do's ✅
- Show real app UI with sample data
- Use consistent device frames (optional but professional)
- Add short, impactful captions
- Highlight key features with annotations
- Use high contrast, readable text
- Show the app's value proposition quickly

### Don'ts ❌
- Don't use placeholder or lorem ipsum text
- Don't show empty states
- Don't include sensitive/personal data
- Don't use low-resolution images
- Don't overcrowd with too much text
- Don't show login/signup screens (unless it's a key feature)

---

## Creating Screenshots

### Method 1: Device Screenshots
1. Run the app on a device/emulator
2. Navigate to the screen you want to capture
3. Take a screenshot (Power + Volume Down on Android)
4. Resize to exact dimensions if needed

### Method 2: Design Tool Mockups
1. Export screens from your design files
2. Place them in device mockups
3. Add captions and annotations
4. Export at correct dimensions

### Method 3: Screenshot Generator Tools
- [AppLaunchpad](https://theapplaunchpad.com)
- [Screenshot Designer](https://screenshotdesigner.com)
- [Previewed](https://previewed.app)
